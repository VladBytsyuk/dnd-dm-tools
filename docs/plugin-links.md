# Ссылки плагина в заметках

Ссылки плагина позволяют открывать сущности dnd-dm-tools прямо из обычного текста заметки. При клике сущность открывается в **«Помощнике ДМа»**.

## Синтаксис

Используйте обычную Markdown-ссылку с протоколом `dnd:`:

```markdown
[алиас](dnd:/путь/к/сущности)
```

Текст в квадратных скобках отображается в заметке как алиас. Адрес после `dnd:` должен совпадать с URL сущности в базе плагина.

## Примеры

```markdown
[Fireball](dnd:/spells/fireball)
[Гоблин](dnd:/bestiary/goblin)
[Длинный меч](dnd:/weapons/longsword)
[Кольцо защиты](dnd:/items/magic/ring_of_protection)
```

## Поддерживаемые разделы

| Раздел | Префикс URL |
| --- | --- |
| Бестиарий | `/bestiary/` |
| Заклинания | `/spells/` |
| Ширма | `/screens/` |
| Оружие | `/weapons/` |
| Доспехи | `/armors/` |
| Предыстории | `/backgrounds/` |
| Черты | `/feats/` |
| Расы | `/races/` |
| Классы | `/classes/` |
| Листы персонажей | `/character-sheets/` |
| Магические предметы | `/items/magic/` |
| Экипировка | `/items/` |

## Примечания

- Ссылки не заменяют обычные ссылки Obsidian и не используют синтаксис `[[...]]`.
- Если ссылка открывается как внешний протокол `dnd:/`, перезагрузите плагин после сборки или обновления.
- Если сущность не найдена по URL, вкладка в «Помощнике ДМа» не откроется.
